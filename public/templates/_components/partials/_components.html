{% load component_library_tags %} {% comment %} public/templates/_components/partials/_components.html — Components panel. Rendered for any FoundationCategory with kind="components". Iterates the category's ``variants`` tuple and includes the category's ``partial`` once per variant under a captioned card. The custom ``include_variant`` tag merges ``variant.context`` into the parent template context so each partial reads its own keys — the dispatch template stays component- agnostic and works for masthead, day-windows, etc. as they land. Same template runs in both the light section and the .dark-wrapped section of the parent ``_panel.html`` shell, so the rendered partial inherits whichever theme cascades from above. Layout ------ ``category.panel_layout`` toggles the wrapper grid: * ``"stack"`` (default) — vertical single-column stack at every breakpoint. * ``"two-col"`` — single column on mobile, two columns on ≥md. Variants flagged with ``solo=True`` span the full row on the two-column layout, so an unpaired entry (e.g. the no-snapshot fallback) doesn't offset every subsequent day/night pair by one column.
